# UVA_flight_control_system：基于 AirSim 的无人机智能控制系统
## 1. 项目背景与优化动机
### 1.1 功能定位
UVA_flight_control_system 是一个基于 AirSim 仿真平台开发的无人机（UAV）核心控制模块。该模块作为无人机系统的 “大脑”，负责处理从底层 API 通信到高层自主航迹生成的全链路逻辑。本项目主要面向无人机算法验证、飞行控制逻辑研究以及基于强化学习的感知避障任务。
本次优化模块归属 OpenHUTB/nn 开源项目，适配课程仿真教学、无人系统控制演示、自主轨迹算法验证等落地场景。
#### 1.1.1 模块在系统中的位置
在无人机技术栈中，本模块处于飞行控制与任务调度层：

上游（环境层）：AirSim 物理仿真引擎（基于 Unreal Engine 4/5），提供重力、风阻及碰撞物理反馈
本模块（控制层）：键盘输入映射、多线程轨迹规划、飞行状态平滑处理、指令解析与调度
下游（执行层）：通过 AirSim API 下发 moveByVelocity、moveToPosition 控制指令，驱动无人机运动执行

#### 1.1.2 核心任务描述

实时交互控制：实现低延迟键盘遥控，支持前后、左右、垂直升降全维度操控
自动化航迹算法：内置多种数学模型，实现环绕、方形、螺旋上升等三维复杂轨迹飞行
动态性能调节：设计多档位速度切换机制，适配精细操控与快速巡航不同需求
应急保障机制：集成一键悬停、一键返航、安全降落逻辑，保障仿真飞行稳定性

#### 1.1.3 应用场景与重要性

仿真教学场景：为智能无人系统课程提供可复现、可调试的无人机控制案例
算法验证场景：快速测试轨迹规划、速度调节、多模式切换等控制算法效果
仿真演示场景：直观展示手动操控与自主飞行结合的混合控制方案

#### 1.1.4 原模块设计缺陷
原生 AirSim 基础示例代码仅实现简单移动功能，飞行操控生硬、无平滑处理，缺少速度分级、应急保护与多轨迹模式，代码耦合度高、无模块化设计，不满足课程作业演示与工程化运行要求。
### 1.2 优化动机
原生示例控制逻辑存在多项短板，本次围绕操控体验、运行稳定性、功能完整性完成全方位优化：

控制平滑度优化原生 API 直接调用会造成无人机启停剧烈抖动，本次引入平滑系数与速度倍率矩阵，模拟实体飞行器惯性特性，飞行轨迹更加自然流畅。

异步多线程改造环绕、方形、螺旋等自动轨迹任务若运行在主线程，会阻塞键盘手动操控。通过 threading 多线程异步执行轨迹算法，实现自主飞行与手动介入无缝切换。

功能模块化与交互优化将复杂底层 API 封装为简易键盘触发指令，新增档位调速、一键悬停、返航、安全退出功能，降低操作门槛，提升演示调试效率。

代码冗余清理删除未定义无效函数、冗余逻辑，统一代码运行链路，解决程序报错、按键卡死、机身掉高漂移等问题，提升环境复现性。

## 4. 针对性功能实现方案
### 4.1 多模式自动飞行状态机
依托键盘按键触发不同飞行模式，通过状态机管理任务切换，互不冲突。
#### 4.1.1 环绕飞行（Orbit）
以无人机起飞原点为圆心，固定环绕半径，动态更新位置与偏航角度，实现 360° 全景环绕观测，运动轨迹连续平滑。
#### 4.1.2 方形巡航（Square）
预设矩形路径关键点，按顺序分段执行直线移动，完成闭合方形航线，适用于固定区域巡逻仿真。
#### 4.1.3 螺旋上升（Spiral）
平面圆周运动叠加垂直缓慢爬升，控制角度递增频率与高度增量，实现稳定三维螺旋轨迹，满足空间侦察仿真需求。
#### 4.1.4 原地旋转
控制机体偏航角匀速变化，实现原地定点旋转，适配姿态调试与环视观测场景。
### 4.2 三级速度分级控制系统
内置低速、标准、高速三档固定倍率，按下 P 键循环切换，控制台同步文字提示当前模式：

低速 0.6 倍：精细微调、近距离精准操控
标准 1.0 倍：日常常规飞行演示
高速 1.6 倍：大范围快速转场巡航全程适配 WASD 全维度移动，变速即时生效，无需中断飞行。

### 4.3 操控稳定性优化
重构键盘监听逻辑，修正速度计算代码位置，避免按键触发高度异常指令；删除未定义函数调用start_eight()，彻底解决程序卡死、按键失灵、莫名掉高漂移问题。
### 4.4 安全保障逻辑设计

一键悬停（H）：调用官方悬停接口，立即终止所有自动任务，锁定当前位置
一键返航（B）：自动回归初始原点坐标，快速复位场景
优雅退出（ESC）：自动执行平稳降落，断开 API 连接，避免仿真器无人机卡死异常
